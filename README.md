# CollectionManager

CollectionManager 是一个面向 osu! 玩家使用的本地桌面工具，用来整理、搜索、合并和导出收藏夹。

它会读取 osu! 目录中的 `osu!.db` 和 `collection.db`，把谱面与收藏夹缓存到本地数据库中，方便你在图形界面里完成批量管理，而不必忍受 osu!(stable) 原生的界面和操作限制。

## 它能做什么

- 导入 osu! 目录中的谱面信息与收藏夹
- 新建、重命名、删除、合并收藏夹
- 像在 osu! 一样搜索谱面，并按标题、艺术家、难度、谱师、最后更新时间排序
- 从当前收藏夹移除谱面
- 打开独立的谱面列表窗口，把谱面批量添加到一个或多个收藏夹
- 使用多选模式批量处理收藏夹或谱面
- 将选中的收藏夹或全部收藏夹导出为标准 `collection.db`
- 从已有的 `collection.db` 导入收藏夹
- 支持中文 / English 界面切换
- 启动时可直接复用之前生成的本地数据库，减少重复导入时间

## 适用场景

- 想批量整理大量 osu! 收藏夹
- 想把多个收藏夹合并成一个新的收藏夹
- 想从全量谱面列表里快速挑选谱面加入收藏夹
- 想导出部分收藏夹，而不是整份 `collection.db`
- 想在不直接修改 osu! 原始数据库文件的前提下完成管理工作

## 使用前准备

运行本程序前，你需要准备：

- Windows 环境
- 已安装 osu!
- 可访问的 osu! 游戏目录
- 目录中至少存在 `osu!.db`

如果同目录下还存在 `collection.db`，程序会一并导入已有收藏夹；如果不存在，也仍然可以先加载谱面数据库，再在程序里新建收藏夹。

## 启动方式

### 方式 1：直接运行可执行文件

从 [Releases](https://github.com/al1-1s/CollectionManager/releases) 下载预编译的可执行文件，双击运行即可。这是最适合普通用户的启动方式，不需要手动安装 Python 依赖。

### 方式 2：通过源码运行

如果你拿到的是源码，而不是打包后的程序，可以这样运行：

```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python -m src.CollectionManager.app.main
```

## 首次启动

程序启动后会先显示一个启动窗口。

1. 选择 osu! 游戏目录
2. 程序会读取 `osu!.db`
3. 如果检测到 `collection.db`，也会同时导入收藏夹
4. 导入完成后打开主窗口

如果你之前已经使用过本程序，并且本地缓存数据库仍然存在，可以勾选“使用先前的数据库”直接进入主界面，跳过重新导入。

## 基本使用

### 管理收藏夹

- 在左侧列表查看所有收藏夹
- 底部输入名称后可以新建收藏夹
- 选中单个收藏夹后可以重命名
- 进入多选模式后可以批量删除或合并多个收藏夹
- 可以导出选中的收藏夹，也可以一键导出全部收藏夹

### 管理收藏夹中的谱面

- 选中左侧收藏夹后，右侧会显示该收藏夹中的谱面
- 可按关键词搜索当前收藏夹中的谱面
- 可按标题、艺术家、难度、谱师、最后更新时间排序
- 在多选模式下可批量移除谱面

### 从全量谱面列表添加谱面

通过菜单打开“谱面列表”窗口后，你可以：

- 浏览当前数据库中的全部谱面
- 搜索与筛选目标谱面
- 选择多首谱面后批量添加到收藏夹
- 将谱面拖放到左侧收藏夹列表中完成添加

### 导入与导出

- 导出：把选中的收藏夹或全部收藏夹保存为标准 `collection.db`
- 导入：把已有的 `collection.db` 再次导入到当前数据库中

这意味着你可以把整理后的结果输出为 osu! 能识别的格式，再自行备份或替换使用。

## 数据与日志

程序运行后会生成两类本地文件：

- `data/`：本地缓存数据库，默认位于程序当前工作目录下
- `logs/`：运行日志。源码运行时默认写入项目根目录下的 `logs/`，打包运行时默认写入可执行文件同级的 `logs/`

如果程序异常退出、导入失败或界面行为不符合预期，优先查看 `logs/` 中当天生成的日志文件。

## 常见问题

### 我游玩的是 osu!(lazer)，能用这个工具吗？

不行。这个工具专门针对 osu!(stable) 设计，读取的是 stable 版本的数据库文件。osu!(lazer) 的数据结构和存储方式与 stable 不同，因此无法兼容。后续可能会增加对 lazer 的支持。

### 为什么启动时提示缺少 `osu!.db`？

你选择的目录不是 osu! 的实际数据目录，或者该目录下没有 `osu!.db`。请重新选择正确的 osu! 安装目录。

### 为什么没有导入到任何收藏夹？

可能是因为所选目录中不存在 `collection.db`。这种情况下程序仍然可以导入谱面信息，你可以在程序里手动创建和整理收藏夹。

### “使用先前的数据库”为什么不可用或没有效果？

只有在本地缓存数据库已经存在且包含谱面数据时，这个选项才有意义。第一次使用时通常需要先完整导入一次。

### 导出的文件有什么用？

导出的文件是标准格式的 `collection.db`，可以作为备份，也可以用于后续导入到其他环境中使用。

## 注意事项

- 本程序会读取 osu! 原始数据库文件，但日常管理操作基于本地缓存完成
- 重新导入数据库会刷新本地缓存内容
- 建议在大规模整理前先备份你原有的 `collection.db`
